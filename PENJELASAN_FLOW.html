Penjelasan Alur Kerja (Flow) dari Sisi Source Code
serta Daftar & Kegunaan Seluruh File
Tugas Akhir — Pemrograman Berbasis Objek (PBO)
Bahasa: Java (Swing GUI)
Aplikasi ini adalah Aplikasi Ringkasan Buku Otomatis berbasis Java Swing (GUI desktop). Arsitekturnya menerapkan pola MVC sederhana dengan pemisahan paket: core (logika algoritma), io (input/output), history (riwayat), dan ui (tampilan).

BAGIAN 1 — Flow Cara Aplikasi Bekerja

1.1. Titik Mulai (Startup)

java MainApp  →  MainApp.main()
  1. Method MainApp.main() dijalankan.
  2. Mengatur Look & Feel mengikuti OS (UIManager.setSystemLookAndFeelClassName()) supaya tampilan tombol/jendela menyesuaikan Windows. Jika gagal, fallback ke tema default Java (Metal).
  3. Memanggil SwingUtilities.invokeLater(...) → membuat objek MainFrame dan menampilkannya.
Kenapa invokeLater? Semua kode UI Swing wajib berjalan di EDT (Event Dispatch Thread) agar thread-safe. Ini praktik standar Swing.

1.2. Pembentukan Jendela Utama

Saat new MainFrame() dipanggil, konstruktor menjalankan 3 langkah:

  1. buildUi() — menyusun tata letak:
    • NORTH (atas): panel "Pengaturan" → dropdown Metode, dropdown Panjang, field API Token.
    • CENTER (tengah): JSplitPane → kiri = Teks Input, kanan = Ringkasan, dengan tombol besar RINGKAS di bawahnya.
    • SOUTH (bawah): status bar (statusLabel).
  2. wireActions() — memasang event listener ke setiap tombol.
  3. updateApiFieldVisibility() — meng-enable/disable field API Token tergantung metode (jika Rule-based, field token jadi abu-abu/nonaktif).

1.3. Alur Inti: Proses Meringkas (paling penting)

Ini jantung aplikasi. Ketika user menekan tombol RINGKAS:

Tombol RINGKAS
   └─> onSummarize()
         ├─ validasi: teks input tidak boleh kosong
         ├─ buildSummarizer()                ← pilih algoritma
         │     ├─ jika "API-based"  → new ApiBasedSummarizer(token)
         │     └─ jika "Rule-based" → new RuleBasedSummarizer()
         │
         ├─ disable tombol + set status "Sedang meringkas..."
         │
         └─ SwingWorker (jalan di BACKGROUND thread)
               ├─ doInBackground(): summarizer.summarize(text)
               └─ done()  (kembali ke EDT):
                     ├─ tampilkan hasil ke outputArea
                     ├─ buat SummaryRecord (teks asli, ringkasan, metode, waktu)
                     ├─ historyManager.add(record)
                     └─ update status bar

Poin kunci desain

  • Polymorphism: buildSummarizer() mengembalikan tipe Summarizer (interface). onSummarize() cukup memanggil summarizer.summarize(text) tanpa peduli implementasi konkretnya rule-based atau API. Inilah inti OOP yang ditonjolkan.
  • SwingWorker: proses meringkas (khususnya panggilan API yang lambat) dijalankan di thread terpisah supaya GUI tidak freeze/macet. Hasil baru ditampilkan saat done().

Dua Jalur Algoritma Summarize

Jalur A — Rule-based (offline, tanpa internet):

  1. Pecah teks jadi kalimat (splitSentences, diwarisi dari AbstractSummarizer).
  2. Hitung frekuensi tiap kata, abaikan stopword (kata umum spt "yang", "the") & kata < 3 huruf.
  3. Normalisasi frekuensi ke skala 0–1.
  4. Skor tiap kalimat = jumlah bobot kata penyusunnya.
  5. Ambil N kalimat skor tertinggi (N dari pilihan Panjang: 2/4/6), kembalikan sesuai urutan asli agar tetap koheren.

Jalur B — API-based (pakai AI HuggingFace):

  1. Validasi teks & token API.
  2. Tentukan min_length/max_length berdasarkan pilihan Panjang.
  3. Susun JSON payload manual, kirim HTTP POST ke endpoint model facebook/bart-large-cnn via HttpURLConnection.
  4. Baca respons, tangani error (401 token salah, 503 model loading, dll).
  5. Parse JSON respons dengan regex untuk ambil summary_text.
Kedua summarizer mewarisi AbstractSummarizer (inheritance), berbagi field method dan helper splitSentences(). Jika gagal, melempar SummarizationException (custom exception) yang ditangkap di done() lalu ditampilkan sebagai dialog error.

1.4. Alur Pendukung (Aksi Tombol Lain)

TombolMethodYang Terjadi
Muat FileonLoadFile()Buka JFileChooser → TextInputHandler.loadFromFile() baca .txt UTF-8 → isi inputArea.
Tempel ClipboardonLoadClipboard()TextInputHandler.loadFromClipboard() ambil teks dari clipboard sistem → isi inputArea.
Bersihkan Input(inline)Kosongkan inputArea.

BantuanonHelp()Tampilkan dialog teks petunjuk pemakaian.

Alur Simpan/Export (polymorphism kedua)

onSave() → pilih filter (txt/pdf)
   └─ Exporter exporter = (pdf) ? new PdfExporter() : new TxtExporter();
   └─ exporter.export(formatter.formatForExport(currentRecord), file);

Sama seperti Summarizer, Exporter adalah interface — kode pemanggil seragam, implementasi .txt vs .pdf berbeda. Menariknya, PdfExporter menulis PDF dari nol mengikuti spesifikasi PDF 1.4 (objek Catalog, Pages, Font, content stream) tanpa library eksternal.

Alur Riwayat

onHistory() → new HistoryDialog(this, historyManager)
   ├─ loadRecords(): ambil historyManager.getAllNewestFirst()  (terbaru di atas)
   ├─ klik entri → updateDetail(): tampilkan detail via formatForExport()
   └─ "Bersihkan Riwayat" → historyManager.clear()

SummaryHistoryManager menyimpan list SummaryRecord in-memory (hilang saat aplikasi ditutup).

1.5. Ringkasan Alur Antar-Paket

        ui (MainFrame, HistoryDialog)   ← lapisan tampilan & event
          │ memanggil
   ┌──────┼─────────────┬──────────────┐
   ▼      ▼             ▼              ▼
 core    io           history      (formatter penghubung)
Summarizer  TextInputHandler   SummaryRecord
 ├ RuleBased Exporter           SummaryHistoryManager
 └ ApiBased  ├ TxtExporter
             └ PdfExporter

Aliran data satu siklus penuh: Input (ketik/file/clipboard) → core meringkas → tampil di output → dibungkus jadi SummaryRecord → masuk history → bisa di-export lewat io.

BAGIAN 2 — Daftar Semua File & Kegunaannya

Paket com.ringkasan (root)

FileKegunaan
MainApp.javaEntry point aplikasi (main). Set Look & Feel OS, lalu tampilkan MainFrame di EDT.

Paket core — Logika Bisnis (Algoritma Ringkasan)

FileKegunaan
Summarizer.javaInterface/kontrak semua algoritma ringkasan (summarize(), getName()). Dasar polymorphism.
AbstractSummarizer.javaKelas abstract yang implement Summarizer. Menyimpan field method (encapsulation: protected + setter validasi) & helper bersama splitSentences(). Contoh inheritance.
RuleBasedSummarizer.javaAlgoritma ringkas offline berbasis frekuensi kata + stopword (Indonesia & Inggris). Tidak butuh internet.
ApiBasedSummarizer.javaAlgoritma ringkas via API AI HuggingFace (model facebook/bart-large-cnn). HTTP request + parser JSON manual tanpa library.
SummaryMethod.javaEnum pilihan panjang ringkasan: SHORT(2), MEDIUM(4), DETAILED(6 kalimat). Mengisi dropdown "Panjang".
SummarizationException.javaCustom exception untuk kegagalan proses ringkas (input kosong, API gagal, dll).

Paket io — Input & Output

FileKegunaan
TextInputHandler.javaMembaca teks dari file .txt (UTF-8) dan dari clipboard sistem.
SummaryFormatter.javaMemformat ringkasan: formatForDisplay() (untuk GUI) & formatForExport() (tambah header tanggal/metode/teks asli untuk file).
Exporter.javaInterface kontrak ekspor ke file (export(), getExtension()). Dasar polymorphism kedua.
TxtExporter.javaImplementasi Exporter → tulis file .txt UTF-8.
PdfExporter.javaImplementasi Exporter → tulis file .pdf dari nol (spec PDF 1.4: multi-halaman, word-wrap, font Helvetica) tanpa library eksternal.

Paket history — Riwayat

FileKegunaan
SummaryRecord.javaEntitas immutable (field final) satu entri riwayat: teks asli, hasil ringkasan, metode, timestamp. Contoh encapsulation.
SummaryHistoryManager.javaPengelola koleksi riwayat in-memory (add, getAllNewestFirst, clear, size).

Paket ui — Tampilan (Swing GUI)

FileKegunaan
MainFrame.javaJendela utama. Menyusun seluruh komponen, menangani semua aksi tombol, mengorkestrasi core+io+history. Menjalankan ringkasan via SwingWorker.
HistoryDialog.javaDialog modal riwayat: daftar entri (kiri) + detail (kanan), tombol bersihkan riwayat.
